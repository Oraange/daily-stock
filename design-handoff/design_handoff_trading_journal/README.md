# Handoff: 매매노트 — 초보 투자자용 주식 매매일지 (React)

## Overview
"매매노트"는 초보 투자자가 자신의 매매를 기록하고, 성과와 **감정**을 함께 회고하도록 돕는 웹 서비스입니다. 대시보드·캘린더·매매일지·성과분석·회고 5개 화면과 "새 매매 기록" 입력 모달로 구성됩니다. 톤은 **따뜻하고 친근한 핀테크**, 밝은 화이트 중심이며, 한국 주식 관례에 따라 **상승/수익은 빨강, 하락/손실은 파랑**을 사용합니다.

## About the Design Files
이 번들의 `매매노트.dc.html`은 **HTML로 만든 디자인 레퍼런스**입니다 — 의도한 룩앤필과 동작을 보여주는 프로토타입이며, 그대로 복붙해 쓰는 프로덕션 코드가 아닙니다. 목표는 이 디자인을 **대상 코드베이스의 기존 환경(React 등)과 패턴·라이브러리로 재현**하는 것입니다. 아직 코드베이스가 없다면 React(권장: Vite + React + TypeScript)로 새로 구성해 구현하세요.

> 참고: 원본 프로토타입은 React 기반 커스텀 런타임(`DCLogic` = React 클래스 컴포넌트) 위에서 동작합니다. 따라서 상태 관리 로직(화면 전환, 캘린더 월/주 토글, 주 이동, 모달)은 그대로 `useState` 기반 React로 옮기기 쉽습니다.

## Fidelity
**High-fidelity (hifi)** — 색상·타이포·간격·인터랙션이 최종안 수준입니다. 아래 명세의 hex/px 값을 그대로 재현하세요. 단, 차트(자산추이·도넛·주간 꺾은선·요일별 막대)는 프로토타입에서 순수 SVG로 그렸으므로, 실제 구현에서는 **Recharts / visx / lightweight-charts** 등 코드베이스에 맞는 차트 라이브러리로 대체해도 됩니다(형태·색상만 맞추면 됨).

## Layout (전체 셸)
- 루트: `display:flex; min-height:100vh; background:#fcf1ea`.
- **사이드바**: `width:236px`, 고정(`position:sticky; top:0; height:100vh`), 흰 배경, 우측 1px 보더. 상단 로고(38×38 코랄 그라디언트 사각 + "매"), nav 5개, 하단으로 밀린 "＋ 새 매매 기록"(다크 버튼) + 사용자 카드.
- **메인**: `flex:1`. 헤더(좌: kicker + 페이지 제목 / 우: "2026년 7월" 칩 + 알림 버튼) + 콘텐츠 영역(`padding:22px 34px 44px`).
- **반응형**: 데스크톱/모바일 둘 다. 좁은 화면에서는 사이드바를 하단 탭바 또는 햄버거 드로어로 전환 권장(프로토타입은 데스크톱 레이아웃 기준). KPI/카드 그리드는 `grid-template-columns:repeat(auto-fit,minmax(...))`로 자동 줄바꿈.

## Screens / Views

### 1. 대시보드 (dash) — 기본 화면
- 제목: "안녕하세요, 투자자님 👋", kicker "HOME".
- **KPI 행**(auto-fit grid): ① 총 자산(다크 그라디언트 카드) `₩32,480,000` + `▲ +4,280,000 (+15.2%)` 배지, ② 총 손익 `+4,280,000`, ③ 이번 달 손익 `+1,240,000`, ④ 평균 손익비 `1.8`.
- **차트 행**(1.7fr / 1fr): 좌 "자산 추이(최근 30일)" 코랄 area+line 곡선, 우 "이번 달 매매 성향" 승률 도넛 62%(빨강 링) + 평균수익/손실/손익비.
- **최근 매매**: 4개 행. 각 행 = 종목 이니셜 아바타 + 종목/날짜·구분·수량 + 감정 칩 + 손익(빨/파) + 수익률.

### 2. 캘린더 (cal) — 핵심 인터랙션
- 상단 **세그먼트 토글**: "월간" / "주간 자산추이" (활성 버튼 흰 배경 + 그림자). 우측에 "7월 실현손익 +1,240,000", "거래일 17일" 요약 카드.
- **월간 모드**: 7열 그리드. 요일 헤더(일=빨강, 토=파랑). 각 셀 = `aspect-ratio:1/.86`, 수익일 배경 `#fdecea`/손실일 `#eaf0fd`. 셀 내부: 상단 날짜 + 거래 점(dot), 하단 그날 손익(빨/파 굵게) + 그날 **총 자산**(회색 `~만` 단위). 범례 포함.
- **주간 모드**: 선택한 한 주의 **자산 변동을 이어지는 꺾은선(area+line) 그래프**로 표시(빨강). 요일(일~토)+날짜 x축, 각 점 위에 자산액(만 단위) 라벨. 상단에 주 이동 `‹ / ›` 버튼 + 주 범위, "주간 변동"·"주말 자산" 요약. → **이 월/주 토글이 이 서비스의 핵심 기능**.

### 3. 매매일지 (journal)
- 필터 칩 행: 전체(활성 다크)/수익/손실/매수/매도.
- 테이블(그리드 `1.4fr .7fr .7fr .8fr 1fr .9fr`): 종목(아바타+이름+날짜) / 구분(매수 빨강·매도 파랑) / 수량 / 평단·청산 / 손익·수익률(빨/파) / 감정 칩. 행 hover `#faf9f6`.

### 4. 성과 분석 (analysis)
- 상단 스탯 카드(auto-fit): 승률 62%, 손익비 1.8, 총 매매 84회, 최대 연속 수익 5회, 평균 보유 3.2일, 최대 손실 -234,000.
- 2열: 좌 "감정별 승률"(감정별 가로 막대, 각 감정색), 우 "요일별 손익"(월~금 막대, +빨강/−파랑).

### 5. 회고 (reflect)
- 상단 하이라이트 카드(코랄 그라디언트 배경 `#fff2ec→#fdeee6`, 보더 `#ffddcf`): "🌱 이번 주 회고" 요약 문구.
- 매매별 회고 카드 리스트: 감정 칩 + 종목 + 날짜 + 수익률, 하단에 메모 인용(연한 면 + 좌측 3px 손익색 보더).

### 6. 새 매매 기록 모달
- 중앙 모달(`min(520px,92vw)`, radius 24). 필드: 종목명 / 구분(매수·매도 세그먼트, 매수 선택 시 빨강) + 수량 / 매수가·청산가 / 감정 선택 칩(선택 시 채워짐) / 메모 textarea. 하단 "취소" + "기록 저장하기"(다크). 오버레이 클릭 시 닫힘, 내부 클릭은 `stopPropagation`.

## Interactions & Behavior
- **화면 전환**: 사이드바 nav 클릭 → 활성 화면 state 변경(활성 항목 `#fff2ec`/코랄 텍스트). 대시보드 "전체보기 →" → 매매일지로 이동.
- **캘린더 토글**: `calMode: 'month' | 'week'`.
- **주 이동**: `week` 인덱스 ±1 (0 ~ 마지막 주 clamp).
- **모달**: `modalOpen` boolean. 사이드바 "＋ 새 매매 기록" / 취소·저장·오버레이로 토글.
- 버튼 hover: 프라이머리 다크 `#211f1b→#37342e`, 인풋 focus 시 보더 `#ff6b4a` + 배경 흰색.

## State Management
```ts
type Screen = 'dash' | 'cal' | 'journal' | 'analysis' | 'reflect';
const [screen, setScreen] = useState<Screen>('dash');
const [calMode, setCalMode] = useState<'month'|'week'>('month');
const [week, setWeek] = useState(1);        // 표시 중인 주 인덱스
const [modalOpen, setModalOpen] = useState(false);
```
- **데이터**: 프로토타입은 정적 샘플(월별 일자 데이터는 `2026-07` 기준으로 시드 함수로 생성, 총자산 누적). 실제 구현에서는 매매 기록 목록에서 파생 계산(일별 손익·누적 자산, 승률, 손익비, 감정별/요일별 집계)하세요. 통화는 KRW, `toLocaleString('en-US')`로 천단위 콤마.
- 캘린더 월 그리드: 해당 월 1일 요일만큼 앞을 `null` 패딩 → 7개씩 주 단위로 chunk. 주간 그래프는 같은 주 배열의 `asset` 값을 이어 그림(비거래일도 직전 자산 유지하여 선이 끊기지 않음).

## Assets
- 별도 이미지 에셋 없음. 로고는 CSS 그라디언트 사각형 + "매" 텍스트. 아이콘·아바타는 이모지 및 종목명 첫 글자. 실제 서비스에서는 아이콘 세트(예: Lucide)와 실제 로고로 교체 권장. 차트는 SVG로 자체 구현되어 있음(라이브러리 대체 가능).
